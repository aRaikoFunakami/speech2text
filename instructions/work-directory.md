# work/ ディレクトリの使い方

## 概要

`work/` は **Git 管理外** の作業用ディレクトリ。
計画書・調査メモ・一時的なスクラッチファイルをここに置く。

---

## 用途

| 用途 | 例 |
|------|-----|
| 開発計画書 | `work/plan-<feature>.md` |
| 調査メモ | `work/research-<topic>.md` |
| 一時的なテストデータ | `work/test-data/` |
| スクラッチコード | `work/scratch.py` |

---

## ルール

1. `work/` は `.gitignore` に登録済み — コミットされない
2. **計画書を `work/` に作成中は実装に着手しない** — ユーザーの承認を得てから開始する
3. 計画書のファイル名は `plan-<機能名>.md` とする
4. 承認後、計画書は参照用としてそのまま残してよい
5. 機密情報（API キー等）を `work/` に保存しない（`.gitignore` でも事故防止のため）
